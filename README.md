# Rising Waters ‚Äî A Machine Learning Approach to Flood Prediction

Floods are among the most devastating natural disasters, and conventional
forecasting often leaves too little time to respond. **Rising Waters** is a
machine-learning powered flood early-warning system: it trains and compares
four classification algorithms on meteorological data, saves the
best-performing model, and serves predictions through a Flask web
application.

## Scenarios this project supports

1. **Early Flood Warning & Evacuation Planning** ‚Äî a meteorologist enters
   current rainfall and cloud visibility readings and gets a risk call hours
   ahead of impact.
2. **Disaster Response & Resource Allocation** ‚Äî a relief coordinator
   monitors several regions at once during monsoon season and prioritises
   resource deployment by instant risk classification.
3. **Model Validation & Performance Assessment** ‚Äî an analyst checks the
   model against historical flood data to confirm reliability for
   operational use.

## Architecture

```
User Browser  ‚Üí  Flask (routing, form validation, preprocessing)
                     ‚Üí  Trained ML Model (model.pkl) + Scaler (scaler.pkl)
                            ‚Üí  Dataset (collection ‚Üí cleaning ‚Üí feature
                               engineering ‚Üí train/test split ‚Üí training)
```

Deployment path: local development ‚Üí Git push ‚Üí GitHub ‚Üí deploy to cloud
(IBM Cloud) ‚Üí live public application.

## Setup

### 1. Environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Dataset

A synthetic dataset generator is included so the whole pipeline runs
end-to-end out of the box:

```bash
python scripts/generate_dataset.py
```

> To use real data instead, download a flood-prediction dataset from Kaggle
> and place it at `data/flood_dataset.csv` with the same column names used
> in `scripts/train_model.py` (`AnnualRainfall`, `MonsoonRainfall`,
> `CloudVisibility`, `Humidity`, `Temperature`, `RiverWaterLevel`,
> `SoilSaturation`, `WindSpeed`, `Elevation`, `DrainageQuality`,
> `FloodOccurred`), or adjust `FEATURE_COLUMNS` to match your data.

### 3. Exploratory data analysis (optional)

```bash
python scripts/data_analysis.py
```

Generates distribution plots, box plots, a correlation heat map, and class
balance charts in `reports/`.

### 4. Train and select the best model

```bash
python scripts/train_model.py
```

Trains Decision Tree, Random Forest, KNN, and XGBoost, prints a confusion
matrix and classification report for each, and saves the best-performing
model + scaler to `model/`.

> **Note:** On the included synthetic dataset, results will differ from a
> real dataset ‚Äî the project brief cites XGBoost at 96.55% accuracy on real
> historical flood data. Swap in the real dataset (step 2) to reproduce
> that result; the training script itself already supports it unchanged.

### 5. Run the web app

```bash
python app.py
```

Open `http://127.0.0.1:5000` in your browser. The app has three pages:

- **Dashboard** ‚Äî project overview and current model accuracy
- **Predict** ‚Äî enter weather readings and get a Flood / No-Flood call
- **History** ‚Äî past predictions made in this session

## Tech stack

- **ML:** NumPy, Pandas, Scikit-learn, XGBoost, Matplotlib, Seaborn
- **Web:** Flask, HTML/CSS
- **Deployment target:** IBM Cloud (or any Python-friendly host ‚Äî Render,
  Railway, PythonAnywhere, etc.)

## Deploying

Any standard Flask deployment works. For a quick free option:

1. Push this repo to GitHub.
2. Connect it to Render / Railway / PythonAnywhere, or provision an IBM
   Cloud Code Engine / Cloud Foundry app.
3. Set the start command to `python app.py` (or use `gunicorn app:app` in
   production) and ensure `requirements.txt` is installed on deploy.
4. Add the live URL and the GitHub repo link to your project page for
   mentor review.
